# Changesets

This directory contains pending changesets for unpublished changes.

## Workflow

1. **After making changes**, run:
   ```bash
   pnpm changeset
   ```
   This creates a markdown file in this directory describing what changed and the bump type (patch/minor/major).

2. **To release**, run:
   ```bash
   pnpm changeset version   # bumps all package.json versions + generates CHANGELOG.md
   pnpm release             # builds + publishes
   ```

## Bump guides

| Change type | Bump |
|---|---|
| Bug fix, dependency update | `patch` |
| New component or feature (backwards-compatible) | `minor` |
| Breaking prop rename, removed export | `major` |

## Per-package CHANGELOG.md

Each publishable package will have a `CHANGELOG.md` auto-generated by `changeset version`. Do not edit CHANGELOG files by hand.
